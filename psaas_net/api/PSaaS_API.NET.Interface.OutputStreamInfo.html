Url

The URL to stream the output files to.

Declaration
public string Url { get; set; }
Property Value
Type Description
System.String
Remarks

Currently only a single URL scheme is supported, MQTT.

The URL supports a few parameters that will be replaced by runtime values. Parameters should be surrounded with curly brackets ({, }).

Parameter Description
jobid The unique ID of the running job returned when the job is started.
filename The name of the file that is being transferred.
  • MQTT

To stream files to MQTT the URL scheme must begin with either mqtt:// or mqtts://. For example:

mqtt://url.com/server/path:port#topic/path/to/use

Where topic/path/to/use should be replaced with the MQTT topic that you want the files sent to. The port should be the port that your MQTT server is listening on or left absent to use the default of 1883 for mqtt and 8883 for mqtts.

Example: This will send files over MQTT to a topic with the job ID and filename in it.

mqtt://url.com/server/path:port#psaas/jobs/{jobid}/{filename}

If the filename is not included as a parameter in the topic it will be sent as a separate message before the actual file is transferred.
